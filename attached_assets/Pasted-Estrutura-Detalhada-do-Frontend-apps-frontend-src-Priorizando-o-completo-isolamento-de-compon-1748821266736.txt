Explicação da Estrutura Frontend Detalhada (apps/frontend/src/modules/):

Isolamento Real: Cada diretório dentro de modules/ (ex: Inbox/, Contacts/, Campaigns/) representa uma feature ou domínio principal da aplicação.
components/: Contém componentes React específicos para aquele módulo.
hooks/: Custom Hooks com lógica de estado e efeitos colaterais relacionados ao módulo.
services/: Funções para interagir com a API backend, específicas para as necessidades do módulo.
pages/: Componentes de página que são renderizados pelas rotas, geralmente compondo os componentes e hooks do módulo.
types/: Definições de tipos TypeScript específicas do módulo.
index.tsx: Ponto de entrada e exportação do módulo, facilitando importações limpas.
shared/: Contém código verdadeiramente global, utilizado por múltiplos módulos.
ui/: Componentes de UI puros e reutilizáveis (Botões, Modais, etc.), idealmente importados do packages/ui se um design system compartilhado for usado.
sockets/: Configuração e gerenciamento do cliente Socket.IO.
lib/: Funções utilitárias genéricas.
config/: Constantes globais, temas, configuração base de API.
contexts/ ou store/: Para gerenciamento de estado global (autenticação, informações do tenant, status da conexão WebSocket) usando Context API ou Zustand.
